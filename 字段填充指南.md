**版本**: v1.0  
**创建日期**: 2026-04-22  
**目标**: 确保不同人填充字段时标准一致

## 一、情绪强度评分标准（1-10分）

### 评分维度

**1-3分（低强度）**:
- 情绪触发较弱，读者可能无感
- 例子：平淡的陈述、客观的描述
- 示例："交易需要学习"（2分）

**4-6分（中等强度）**:
- 情绪触发明显，读者有感觉但不强烈
- 例子：适度的共鸣、轻微的焦虑
- 示例："很多人学交易都会遇到困难"（5分）

**7-8分（高强度）**:
- 情绪触发强烈，读者有明显反应
- 例子：强烈的共鸣、明显的焦虑、愤怒
- 示例："散户总是亏钱，你是不是也这样？"（8分）

**9-10分（极高强度）**:
- 情绪触发极强，读者有强烈情绪反应
- 例子：极度愤怒、极度焦虑、极度震撼
- 示例："别再瞎改缠论了！"（9分）

### 评分技巧

1. **读出声音测试**: 大声读出这段话，感受自己的情绪反应
2. **替换测试**: 如果换成平淡的表述，情绪强度会降低多少？
3. **目标读者测试**: 想象目标读者看到这段话的反应

## 二、情绪类型识别规则

### 核心情绪类型

**1. 焦虑（Anxiety）**
- **特征**: 担心、恐惧、不安
- **触发词**: "亏钱"、"失败"、"被割韭菜"、"走错路"
- **心理需求**: 安全需求未被满足
- **示例**: "散户总是亏钱"、"你是不是也踩过这些坑"

**2. 愤怒（Anger）**
- **特征**: 生气、不满、批判
- **触发词**: "瞎改"、"误导"、"破坏"、"错误"
- **心理需求**: 自尊需求+安全需求未被满足
- **示例**: "别再瞎改缠论了"、"这些做法是在破坏逻辑"

**3. 共鸣（Resonance）**
- **特征**: 认同、理解、归属
- **触发词**: "我也曾经"、"你懂的"、"不是一个人"
- **心理需求**: 归属需求未被满足
- **示例**: "我也曾经这样"、"40岁学交易"

**4. 希望（Hope）**
- **特征**: 乐观、期待、可能性
- **触发词**: "可以做到"、"只要"、"方法"
- **心理需求**: 自我实现需求未被满足
- **示例**: "只要掌握这3点"、"你也可以做到"

**5. 震撼（Shock）**
- **特征**: 惊讶、颠覆认知、反常识
- **触发词**: 精确数据、反常识观点
- **心理需求**: 认知需求未被满足
- **示例**: "99.9%是克制"、"7200秒只操作5秒"

**6. 平静（Calm）**
- **特征**: 安心、稳定、总结
- **触发词**: "记住"、"简单"、"慢慢来"
- **心理需求**: 安全需求被满足
- **示例**: "交易很简单"、"成为内行"

**7. 认同（Identity）**
- **特征**: 身份标签、归属感、价值升华
- **触发词**: "真正的交易者"、"内行"、"修炼"
- **心理需求**: 自尊需求+归属需求未被满足
- **示例**: "真正的交易者"、"修炼的道场"

### 识别技巧

1. **关键词扫描**: 先扫描触发词，快速定位情绪类型
2. **心理需求映射**: 问自己"这段话激活了读者的什么需求？"
3. **情绪强度评估**: 同一情绪类型可能有不同强度

## 三、标题钩子机制分类标准

### 核心钩子机制

**1. 好奇缺口（Curiosity Gap）**
- **定义**: 制造信息缺口，激发好奇心
- **公式**: 提出问题但不给答案
- **示例**: "散户为什么总是亏钱？"
- **强度评估**: 问题越具体、越切中痛点，强度越高

**2. 反常识（Counter-Intuitive）**
- **定义**: 颠覆常识，制造认知冲突
- **公式**: 提出与常识相反的观点
- **示例**: "交易99%时间都在等待？"
- **强度评估**: 反差越大，强度越高

**3. 利益承诺（Benefit Promise）**
- **定义**: 承诺具体利益或结果
- **公式**: 明确告诉读者能得到什么
- **示例**: "我用这4个方法重构了日内交易系统"
- **强度评估**: 利益越具体、越可量化，强度越高

**4. 身份认同（Identity Match）**
- **定义**: 明确目标人群，建立归属感
- **公式**: 用身份标签吸引目标读者
- **示例**: "散户"、"交易者"、"投资者"
- **强度评估**: 身份标签越精准，强度越高

**5. 命令式（Imperative）**
- **定义**: 用命令语气制造紧迫感
- **公式**: "别再...了"、"停止..."、"立即..."
- **示例**: "别再瞎改缠论了！"
- **强度评估**: 命令语气越强烈，强度越高

**6. 数字承诺（Number Promise）**
- **定义**: 用具体数字增强可信度
- **公式**: "3个方法"、"5个步骤"、"7个原则"
- **示例**: "这3个底层逻辑"、"这4个方法"
- **强度评估**: 数字越具体（3-7最佳），强度越高

### 标题公式库（15种）

**F01**: [目标人群]+[痛点]+[解决方案]  
示例: "散户亏钱的3个原因"

**F02**: [反问]+[痛点]+[解决方案]  
示例: "为什么你总是亏钱？"

**F03**: [反问]+[数字]+[底层逻辑]  
示例: "散户为什么总是亏钱？因为你不懂这3个底层逻辑"

**F04**: [反常识数据]+[反问]  
示例: "交易99%时间都在等待？"

**F05**: [反常识数据]+[个人经验]+[方法承诺]  
示例: "交易99%时间都在等待？我用这4个方法重构了日内交易系统"

**F06**: [收藏级]+[干货]+[主题]  
示例: "「收藏级干货」终于有人把"艾略特波浪理论"讲明白了"

**F07**: [深度解读]+[最高级]+[主题]  
示例: "深度解读迄今预测价值最好的交易策略指标：艾略特波浪理论"

**F08**: [比喻]+[价值定位]  
示例: "交易，是一门不求人的手艺"

**F09**: [场景引入]+[问题提出]  
示例: "在股市活下来的人，都练成了这种"可怕"的本事"

**F10**: [能力分析]+[结果承诺]  
示例: "具备这三种能力，你才能在股市上赚到钱"

**F11**: [条件句]+[结果承诺]  
示例: "如果你交易小资金账户，一定要认真看这个！！！"

**F12**: [命令式]+[批判]+[权威背书]  
示例: "别再瞎改缠论了！原版缠论才是最强交易法则"

**F13**: [数据震撼]+[方法承诺]  
示例: "期货股票交易做到这5点，说明你快要成功了！"

**F14**: [从零]+[实践教学]  
示例: "从零构建A股量化交易工具：基于Qlib的全栈系统指南"

**F15**: [技术名称]+[策略类型]  
示例: "双均线缠论策略"

## 四、开头钩子强度评分标准

### 评分维度（1-10分）

**1. 共鸣度（Relatability）**
- 1-3分: 读者无法产生共鸣
- 4-6分: 部分读者有共鸣
- 7-8分: 大部分读者有共鸣
- 9-10分: 几乎所有目标读者都有强烈共鸣

**2. 好奇心激发（Curiosity）**
- 1-3分: 无法激发好奇心
- 4-6分: 轻微激发好奇心
- 7-8分: 明显激发好奇心
- 9-10分: 强烈激发好奇心，必须继续读

**3. 情绪冲击力（Emotional Impact）**
- 1-3分: 情绪冲击弱
- 4-6分: 情绪冲击中等
- 7-8分: 情绪冲击强
- 9-10分: 情绪冲击极强

**4. 清晰度（Clarity）**
- 1-3分: 表达模糊，不知所云
- 4-6分: 表达基本清晰
- 7-8分: 表达清晰，易懂
- 9-10分: 表达极其清晰，一目了然

### 3秒留存率预测公式

**基础公式**:
```
3秒留存率 = (共鸣度 × 0.3) + (好奇心 × 0.3) + (情绪冲击力 × 0.3) + (清晰度 × 0.1)
```

**调整系数**:
- 数据震撼型: +5%
- 个人故事型: +3%
- 批判式: +5%
- 平淡陈述型: -10%

**示例计算**:
- 共鸣度: 9分
- 好奇心: 7分
- 情绪冲击力: 8分
- 清晰度: 8分
- 类型: 个人故事型

计算: (9×0.3) + (7×0.3) + (8×0.3) + (8×0.1) + 3% = 8.0 + 3% = 83%

## 五、可迁移性判断标准

### 判断维度

**1. 钩子机制的通用性**
- **高通用性**: 数据震撼、反常识、方法承诺
- **中等通用性**: 个人故事、痛点批判
- **低通用性**: 特定领域的专业术语、特定事件

**2. 目标人群的相似性**
- **可迁移**: 目标人群的痛点、需求、心理状态相似
- **不可迁移**: 目标人群的痛点、需求、心理状态差异大

**3. 情绪类型的适配性**
- **可迁移**: 情绪类型适合新主题
- **不可迁移**: 情绪类型不适合新主题

### 判断流程

1. **提取钩子核心机制**: 去掉具体内容，只保留机制
2. **评估机制通用性**: 这个机制是否适用于其他主题？
3. **列举可迁移主题**: 具体列出3-5个可迁移的主题
4. **列举不可迁移主题**: 具体列出3-5个不可迁移的主题
5. **说明原因**: 为什么可以/不可以迁移？

### 示例

**原文**: "交易，是一门不求人的手艺"

**钩子核心机制**: 比喻+身份认同+价值承诺

**可迁移主题**:
- "理财，是一门不求人的手艺"（相同机制）
- "编程，是一门不求人的手艺"（相同机制）
- "写作，是一门不求人的手艺"（相同机制）

**不可迁移主题**:
- "量化交易技术分析"（纯技术主题，不适合比喻）
- "股市暴跌如何应对"（紧急主题，不适合'手艺'的长期主义）

**原因**: 比喻+身份认同机制适合长期主义、技能学习主题，不适合纯技术或紧急主题

## 六、填充工作流程

### Step 1: 通读全文（5分钟）
- 快速通读全文，理解文章主题和结构
- 标注情绪触发点（用荧光笔或注释）
- 识别标题和开头的核心机制

### Step 2: 填充情绪价值架构（20分钟）

**2.1 情绪曲线**:
1. 将文章分成4-5个阶段（每300-400字一个阶段）
2. 为每个阶段命名（痛苦累积、转折、希望、平静等）
3. 标注字数范围
4. 评估情绪强度（1-10分）
5. 提取情绪触发点（3-5个关键词/句子）

**2.2 情绪密度**:
1. 统计每300字的情绪触发点数量
2. 计算平均密度
3. 标注分布情况

**2.3 情绪类型**:
1. 识别文章使用的情绪类型（焦虑、愤怒、共鸣、希望等）
2. 估算每种情绪的占比（加起来=100%）
3. 提取每种情绪的代表性例子（2-3个）
4. 分析心理触发机制

**2.4 心理需求映射**:
1. 为每种情绪映射对应的心理需求
2. 描述需求的具体表现
3. 分析设计意图

### Step 3: 填充标题工程（10分钟）

**3.1 钩子机制**:
1. 识别标题的主要钩子机制（好奇缺口、反常识、利益承诺等）
2. 识别次要钩子机制
3. 匹配标题公式库编号（F01-F15）
4. 分析为什么这个机制有效

**3.2 标题公式**:
1. 提取标题使用的公式
2. 写出公式模板
3. 用原标题作为示例
4. 解释为什么这个公式有效

**3.3 强度评分**:
1. 评估好奇缺口强度（1-10分）
2. 评估利益承诺强度（1-10分）
3. 评估身份认同强度（1-10分）
4. 评估紧迫感强度（1-10分）
5. 计算总分（平均值）

**3.4 关键元素**:
1. 识别目标人群
2. 识别痛点
3. 识别解决方案暗示
4. 识别情绪触发器
5. 识别价值承诺

### Step 4: 填充开头钩子（10分钟）

**4.1 钩子机制**:
1. 识别开头类型（个人故事、痛点批判、数据震撼等）
2. 识别子类型
3. 描述具体技巧
4. 分析为什么这个机制有效

**4.2 强度评分**:
1. 评估共鸣度（1-10分）
2. 评估好奇心激发（1-10分）
3. 评估情绪冲击力（1-10分）
4. 评估清晰度（1-10分）
5. 计算总分（平均值）

**4.3 留存率预测**:
1. 使用公式计算3秒留存率
2. 计算10秒留存率（通常是3秒的80-90%）
3. 说明预测理由

**4.4 可迁移性**:
1. 判断是否可迁移（true/false）
2. 列举可迁移主题（3-5个）
3. 列举不可迁移主题（3-5个）
4. 说明原因
5. 提供迁移示例

**4.5 关键元素**:
1. 提取3-5个关键元素
2. 为每个元素标注内容和功能

### Step 5: 自检（5分钟）

**检查清单**:
- [ ] 所有必填字段都已填充
- [ ] 情绪强度评分合理（不全是10分）
- [ ] 情绪类型占比加起来=100%
- [ ] 标题公式匹配正确
- [ ] 开头钩子可迁移性判断有理有据
- [ ] 所有示例都来自原文
- [ ] YAML格式正确（缩进、冒号、引号）

## 七、常见错误与避免

### 错误1: 情绪强度评分过高
- **表现**: 所有情绪强度都是9-10分
- **原因**: 没有对比参照，主观感觉过强
- **避免**: 先评估最强的情绪（9-10分），再相对评估其他情绪

### 错误2: 情绪类型识别不准
- **表现**: 把"焦虑"误认为"愤怒"
- **原因**: 没有理解情绪的核心特征
- **避免**: 使用关键词扫描法，先找触发词再判断情绪类型

### 错误3: 标题公式匹配错误
- **表现**: 把F03误认为F05
- **原因**: 没有仔细对比公式模板
- **避免**: 先提取标题结构，再逐个对比公式库

### 错误4: 可迁移性判断过于乐观
- **表现**: 认为所有钩子都可以迁移
- **原因**: 没有考虑目标人群和情绪类型的差异
- **避免**: 使用判断流程，逐步评估通用性、相似性、适配性

### 错误5: YAML格式错误
- **表现**: 缩进不对、冒号后没有空格、引号不匹配
- **原因**: 不熟悉YAML语法
- **避免**: 使用YAML验证工具，或参考已有示例

## 八、质量标准

### 优秀（90-100分）
- 所有字段填充完整且准确
- 情绪强度评分有梯度（不全是高分）
- 情绪类型识别准确，占比合理
- 标题公式匹配正确
- 可迁移性判断有理有据，示例具体
- YAML格式完全正确

### 良好（80-89分）
- 所有字段填充完整
- 情绪强度评分基本合理
- 情绪类型识别基本准确
- 标题公式匹配基本正确
- 可迁移性判断基本合理
- YAML格式基本正确

### 及格（70-79分）
- 大部分字段填充完整
- 情绪强度评分有一定参考价值
- 情绪类型识别有部分错误
- 标题公式匹配有误差
- 可迁移性判断较粗糙
- YAML格式有小错误

### 不及格（<70分）
- 字段填充不完整
- 情绪强度评分不合理
- 情绪类型识别错误较多
- 标题公式匹配错误
- 可迁移性判断缺乏依据
- YAML格式错误较多
