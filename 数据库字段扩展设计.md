# 爆款文章数据库字段扩展设计

**设计日期**: 2026-04-22  
**目标**: Phase 1 - 核心引擎（情绪价值架构 + 标题工程 + 开头钩子）

---

## 一、当前字段结构

### Frontmatter 元数据
```yaml
---
title: 文章标题
source: 来源平台
url: 原文链接
author: 作者
publish_date: 发布日期
stats:
  views: 阅读量
  likes: 点赞数
  favorites: 收藏数
  comments: 评论数
keywords: [关键词1, 关键词2, ...]
---
```

### 分析章节
- **开头方式**: 类型 + 金句 + 特点
- **论证结构**: 类型 + 逻辑链 + 特点
- **数据呈现**: 类型 + 核心数据 + 特点
- **金句提取**: 开头/结尾金句列表
- **结尾方式**: 类型 + 特点
- **可复用元素**: 勾选清单
- **写作特色**: 自由文本描述
- **核心论点**: 论点列表
- **宽论植入建议**: 植入位置 + 方式 + 合规要点

## 二、新增字段设计（Phase 1）

### 2.1 情绪价值架构 ⭐⭐⭐⭐⭐

**为什么新增**: 这是爆款的核心引擎。当前系统只保留情绪内容,不分析情绪机制。

**新增字段**:
```yaml
emotional_architecture:
  # 情绪曲线: 文章的情绪流动路径
  emotional_curve:
    - stage: "痛苦累积"
      range: "0-300字"
      intensity: 8  # 1-10分
      triggers: ["散户总是亏钱", "你是不是也这样"]
    - stage: "转折"
      range: "300-500字"
      intensity: 6
      triggers: ["但其实", "真相是"]
    - stage: "希望"
      range: "500-800字"
      intensity: 7
      triggers: ["只要掌握", "你也可以"]
    - stage: "平静"
      range: "800-1000字"
      intensity: 4
      triggers: ["记住", "总结"]
  
  # 情绪密度: 每300字的情绪触发点数量
  emotional_density:
    average: 2.5  # 平均每300字的触发点数
    distribution:
      - range: "0-300字"
        count: 3
      - range: "300-600字"
        count: 2
      - range: "600-900字"
        count: 2
  
  # 情绪类型: 文章使用的情绪类型及占比
  emotional_types:
    - type: "焦虑"
      percentage: 40
      examples: ["散户总是亏钱", "你是不是也踩过这些坑"]
    - type: "共鸣"
      percentage: 30
      examples: ["我也曾经这样", "这种感觉我懂"]
    - type: "希望"
      percentage: 20
      examples: ["只要掌握这3点", "你也可以做到"]
    - type: "平静"
      percentage: 10
      examples: ["记住", "慢慢来"]
  
  # 心理需求映射: 每个情绪对应的未被满足的心理需求
  psychological_needs:
    - emotion: "焦虑"
      need: "安全需求未被满足"
      manifestation: "担心亏钱、害怕被割韭菜"
    - emotion: "共鸣"
      need: "归属需求未被满足"
      manifestation: "感觉自己不是一个人在战斗"
    - emotion: "希望"
      need: "自我实现需求未被满足"
      manifestation: "渴望掌握技能、实现财务自由"
```

**使用场景**:
- 仿写时: 设计相同强度的情绪曲线
- 评估时: 计算情绪曲线相似度 ≥80%

### 2.2 标题工程 ⭐⭐⭐⭐⭐

**为什么新增**: 标题决定80%的打开率。当前系统完全保留标题,但不理解标题为什么有效。

**新增字段**:
```yaml
title_engineering:
  # 标题钩子机制: 标题吸引点击的核心机制
  hook_mechanism:
    primary: "反问+数字+底层逻辑"  # 主要机制
    secondary: ["好奇缺口", "身份认同"]  # 次要机制
    formula_id: "F03"  # 对应标题公式库编号
  
  # 标题公式: 使用的标题公式
  formula:
    name: "反问式+数字+承诺"
    template: "[目标人群]为什么[痛点]？因为你不懂这[数字]个[解决方案]"
    example: "散户为什么总是亏钱？因为你不懂这3个底层逻辑"
  
  # 标题强度评分: 1-10分
  strength_score: 9
  strength_breakdown:
    curiosity_gap: 9  # 好奇缺口强度
    benefit_promise: 8  # 利益承诺强度
    identity_match: 9  # 身份认同强度
    urgency: 6  # 紧迫感强度
  
  # 标题关键元素
  key_elements:
    target_audience: "散户"  # 目标人群
    pain_point: "总是亏钱"  # 痛点
    solution_hint: "3个底层逻辑"  # 解决方案暗示
    emotional_trigger: "反问"  # 情绪触发器
```

**使用场景**:
- 仿写时: 使用相同钩子机制生成新标题
- 评估时: 检查标题钩子机制是否一致

---

### 2.3 开头钩子 ⭐⭐⭐⭐

**为什么新增**: 开头决定读者是否继续阅读。当前系统完全保留第一段,但不评估钩子强度。

**新增字段**:
```yaml
opening_hook:
  # 开头钩子机制: 开头留住读者的核心机制
  hook_mechanism:
    type: "个人故事引入"  # 故事/痛点/反常识/数据/提问
    sub_type: "情感共鸣"
    technique: "从个人与金钱的关系切入"
  
  # 开头钩子强度: 1-10分
  strength_score: 8
  strength_breakdown:
    relatability: 9  # 共鸣度
    curiosity: 7  # 好奇心激发
    emotional_impact: 8  # 情绪冲击力
    clarity: 8  # 清晰度
  
  # 3秒留存率预测: 基于钩子强度的预测
  retention_prediction:
    three_second: 85  # 3秒留存率预测（%）
    ten_second: 70  # 10秒留存率预测（%）
  
  # 开头可迁移性: 评估钩子是否可迁移到新主题
  transferability:
    is_transferable: true
    transferable_to: ["其他个人成长主题", "其他技能学习主题"]
    not_transferable_to: ["纯技术分析", "数据驱动主题"]
    reason: "个人故事钩子依赖情感共鸣,适合成长类主题,不适合纯理性分析"
  
  # 开头关键元素
  key_elements:
    - element: "个人经历"
      content: "40岁学交易"
    - element: "情感共鸣点"
      content: "钱像空气一样存在"
    - element: "冲突设置"
      content: "冷水洗礼"
```

**使用场景**:
- 仿写时: 评估钩子是否可迁移,如果可以则设计类似强度的钩子
- 评估时: 检查开头钩子强度 ≥原文80%

## 四、实施计划

### 4.1 更新现有11篇文章

**工作量估算**: 每篇文章约1小时（分析+填充新字段）

**优先级排序**:
1. 先更新3篇代表性文章（不同结构类型）
2. 验证新字段设计是否合理
3. 调整字段设计（如果需要）
4. 批量更新剩余8篇文章

**代表性文章选择**:
- 个人成长叙事型: `wechat_jiaoyi_交易是不求人的手艺_20260421.md`
- 观点论证型: `wechat_chanlun_别再瞎改缠论了_20260421.md`
- 方法论型: `wechat_jiaoyi_日内交易4方法_20260421.md`

### 4.2 创建字段填充指南

**目标**: 确保不同人填充字段时标准一致

**指南内容**:
- 情绪强度评分标准（1-10分）
- 情绪类型识别规则
- 标题钩子机制分类标准
- 开头钩子强度评分标准
- 可迁移性判断标准

### 4.3 验证标准

**Phase 1.1 完成标准**:
- ✅ 新字段设计文档完成
- ✅ 3篇代表性文章更新完成
- ✅ 字段填充指南完成
- ✅ 新字段设计经过验证,无需调整
- ✅ 剩余8篇文章更新完成
